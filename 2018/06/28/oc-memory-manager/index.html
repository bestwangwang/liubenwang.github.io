读书笔记：内存管理和引用计数

字数统计: 3,223 | 阅读时长 ≈ 14

内存管理的思考方式:

  1. 自己生成并持有对象
  2. 非自己生成的对象，也可被持有
  3. 不再需要自己持有的对象时，需要释放
  4. 非自己持有的对象不能被释放

自己生成并持有对象

值得注意的是：

1.使用alloc/new/copy/mutableCopy生成的对象，持有该对象。

2.默认以alloc/new/copy/mutableCopy开头的方法均可自己生成并持有对象。

3.使用copy/mutableCopy生成对象副本，同样是生成并持有对象，但不影响原对象引用计数。

非自己生成并持有对象

值得注意的是：

1.使用alloc/new/copy/mutableCopy方法以外取得的对象，不是自己生成的对象不持有对象；

2.通过[NSMutableArray array]生成的对象，对象存在但变量ms并不持有对象，不能释放，可通过retain持有对象。

3.autorelease将对象加入自动释放池，使得对象存在但不持有，在超出生存范围后由自动释放池主动调用release释放。

不再需要自己持有的对象需释放

值得注意的是：

1.使用alloc/new/copy/mutableCopy或以此开头的方法生成的对象，在不需要的时候需要释放，否则会造成内存泄漏。

2.自己生成并持有的对象，一旦被release释放便不能再访问，会偶发空指针错误崩溃，即便是再次retain也不能访问。

非自己持有的对象不能被释放

值得注意的是：

1.使用alloc/new/copy/mutableCopy或以此开头的以外的方法生成的对象，不被持有，故不能释放，强行释放会崩溃。

2.通过[NSMutableArray array]生成的对象不被持有，且是交由自动释放池释放，此时释放非自己持有的对象后，当超出该对象生存范围时，自动释放池释放一个已经被释放的对象时会抛出空指针异常。

ARC下的所有权修饰符

ARC有效时，所有类型必须加上所有权修饰符。

  • __strong修饰符
  • __weak修饰符
  • __unsafe_unretained修饰符
  • __autoreleasing 修饰符
__strong修饰符

__strong修饰符是id类型和所有对象类型的默认所有权修饰符。表示对对象的强引用，在变量超出作用域时，强引用失效，对象销毁。

__weak修饰符

__weak修饰符可用来解决循环引用问题，表示弱引用，弱引用不持有对象。

注意：将自己生成并持有的对象赋值给__weak修饰的变量，即持有对自己生成并持有对象的弱引用，自己生成并持有的对象无强引用将立即被销毁

在持有某对象的弱引用时，若该对象被废弃，则该弱引用自动失效并处于nil被赋值状态。

__unsafe_unretained修饰符

__unsafe_unretain修饰符是不安全的修饰符，附有该修饰符的对象不属于编译器的内存管理对象。

__autoreleasing修饰符

附有__autoreleasing修饰符的对象将被加入自动释放池中，ARC下的自动释放池

非自己生成并持有的对象将自动加入自动释放池中

id的指针和对象的指针在没有显示指定时会被赋上__autorelease修饰符。

ARC规则

  • 不能使用retain/release/retainCount/autorelease
  • 不能使用NSAllocObject/NSDeallocObject
  • 必须遵守内存管理的方法命名规则
  • 不能显示调用dealloc
  • 使用@autoreleasepool代替NSAutoreleasePool
  • 不能使用区域（NSZone）
  • 对象型变量不能作为C结构体成员
  • 显式转换id和void *
显式转换id和void *

在MRC中id和void *可以相互转化，但是在ARC中会编译错误。id型或对象型变量赋值给void * 或者逆向赋值时需要进行特定的转换。

使用__birdge进行赋值安全性低，甚至低于__unsafe_retained，如果管理这不注意赋值对象的所有者，就会因为悬垂指针而导致程序崩溃。

__birdge转换中还有另外两种转换，__birdge_retained和__bridge_tranfer：

__birdge_retained转换可使要转换赋值的变量也持有所赋值的对象.

__bridge_tranfer转换可使得被转换的变量所引用的对象在该变量被赋值给目标变量后随之释放。

objc_retainAutoreleaseReturnValue函数主要用于最优化程序运行，自己持有对象函数，和objc_autorelaeaseReturnValue成对使用。

使用objc_autorelaeaseReturnValue用来返回注册到autoreleasepool中的对象，并检查使用该函数的方法或函数调用方的执行命令列表，如果同时调了objc_retainAutoreleaseReturnValue函数，就不讲返回的对象注册到autoreleasepool中，而是直接传递到方法或者函数的调用发。两者配合使用可以不将对象注册到autoreleasepool中而直接传递，达到最优化。

对象被废弃时动作如下：
1.从weak表中获取被销毁对象的地址为键值的记录;

2.将包含在记录中的所有附有__weak修饰符变量的地址赋值为nil;

3.从weak表中删除该记录

4.从引用计数表中删除废弃的对象的地址为键值的记录。
